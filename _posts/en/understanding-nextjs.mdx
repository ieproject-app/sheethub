## What are Server Components?
Server Components are a new feature in Next.js that allow you to write UI components that run exclusively on the server. This can lead to significant performance improvements by reducing the amount of JavaScript sent to the client.

> Note: Server Components can directly access server-side resources like databases or file systems without needing to create a separate API route. This simplifies data fetching logic immensely.

Here's a long example of what a Server Component might look like, fetching data directly:
```tsx
// src/app/page.tsx
import { db } from '@/lib/db';
 
// This is a Server Component
async function HomePage() {
  const posts = await db.post.findMany();
 
  return (
    <main>
      <h1>My Blog</h1>
      <ul>
        {posts.map((post) => (
          <li key={post.id}>
            <a href={`/posts/${post.slug}`}>{post.title}</a>
          </li>
        ))}
      </ul>
    </main>
  );
}

export default HomePage;
```

## When to use Client Components?
Client Components are the components you're already familiar with from traditional React applications. They are rendered on the client and can use state, effects, and browser-only APIs. You opt into them by using the `'use client'` directive at the top of a file.

- They can use `useState` and `useEffect`.
- They have access to browser APIs like `window` or `localStorage`.
- They are ideal for interactive UI elements.

> Remember that you should only use Client Components when you absolutely need them. Pushing as much logic as possible to the server is the Next.js way.
