I recently decided to perform a clean installation of Windows 11 LTSC edition. If you are not familiar, LTSC is a lightweight version of Windows that strips away most of the bloatware, including Microsoft's built-in apps like OneDrive. It is great for performance, but strangely, once I reached the desktop, I found two "ghost" OneDrive folders hanging out in my File Explorer navigation pane.

These folders were essentially useless. They couldn't be deleted through the UI, and clicking them did absolutely nothing. It is a small detail, but for someone who likes a clean workspace, it was incredibly annoying. After trying standard fixes like unlinking an account (which wasn't even there), I found the solution hidden deep within the Windows Registry.

<Callout variant="warning" title="Critical Warning: Backup Your Registry">
  This method involves editing the Windows Registry. Errors here can cause serious system issues. Proceed at your own risk, and I highly recommend [backing up your registry](https://support.microsoft.com/en-us/topic/how-to-back-up-and-restore-the-registry-in-windows-855140ad-e318-2a13-2829-d428a2ab0692) before starting.
</Callout>

## Why This Happens (Quick Context)

In LTSC or custom debloated installs, OneDrive app components may be removed but some Explorer registration keys can remain. File Explorer reads those registration keys (CLSID/NameSpace), then still renders a OneDrive node even though the app endpoint is gone.

That is why you see a folder in the sidebar that does nothing when clicked.

## Safe Registry Backup (Do This First)

Before changing anything:

<Steps>
  <Step>
    **Create a System Restore Point**
    Open Start, search for **Create a restore point**, then create one with a clear label.
  </Step>

  <Step>
    **Export only the key you are about to edit**
    In Registry Editor, right-click the target key and choose **Export**.
  </Step>

  <Step>
    **Store backup `.reg` files in a dedicated folder**
    Keep one folder per fix session so rollback is fast and unambiguous.
  </Step>
</Steps>

## Part 1: Removing Folder Entries from the NameSpace

The first step is to clean up the 'keys' that tell File Explorer to display these folders in the sidebar.

<Steps>
  <Step>
    ### Open Registry Editor
    Press <kbd>Win</kbd> + <kbd>R</kbd>, type `regedit`, and press <kbd>Enter</kbd>.
  </Step>

  <Step>
    ### Navigate to NameSpace
    Copy and paste the following address into the top bar and press <kbd>Enter</kbd>:
    
    `Computer\HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Desktop\NameSpace`

    ![Navigate to the NameSpace path in the Registry Editor](/images/_posts/windows11/onedrive-duplicate/onedrive-2.webp)
    *Double-check you are in the HKEY_CURRENT_USER hive.*
  </Step>

  <Step>
    ### Identify and Delete OneDrive Keys
    Under the `NameSpace` folder in the left pane, you'll see several folders with long alphanumeric names (CLSIDs). Click each one and check the **Data** column in the right pane.
    
    Look for entries that mention **"OneDrive"**. Once found, **right-click** that folder in the left pane and select **Delete**. Repeat this for any duplicate OneDrive entries you find.

    ![Delete the registry key folder related to OneDrive](/images/_posts/windows11/onedrive-duplicate/onedrive-3.webp)
    *Removing the CLSID folder associated with OneDrive.*
  </Step>
</Steps>

Close the Registry Editor and reopen File Explorer. In many cases, those duplicate folders will vanish immediately.

## Part 2: Eliminating the Stubborn "OneDrive - Personal" Folder

If "OneDrive - Personal" still reappears or refuses to go away after a restart, we need to adjust one specific stubborn key.

This second step works because `System.IsPinnedToNameSpaceTree` controls whether that shell folder is pinned in Explorer navigation.

<Steps>
  <Step>
    ### Locate the Specific CLSID
    Back in Registry Editor, navigate to this exact path:
    
    `Computer\HKEY_CLASSES_ROOT\CLSID\{018D5C66-4533-4307-9B53-224DE2ED1FE6}`
  </Step>

  <Step>
    ### Change Pinning Status
    In the right-hand pane, find the entry named `System.IsPinnedToNameSpaceTree`. **Double-click** it and change the **Value data** from `1` to `0`. Click **OK**.

    ![Changing the pinning value to 0](/images/_posts/windows11/onedrive-duplicate/onedrive-5.webp)
    *Setting this to 0 prevents it from being pinned to the sidebar.*
  </Step>
</Steps>

You might need to restart your computer for the changes to take full effect. Once done, your File Explorer should be back to a clean, minimal state.

## Recovery (If Something Looks Wrong)

If Explorer behaves unexpectedly after edits:

1. Double-click your exported `.reg` backup file to restore the original key.
2. Restart Explorer from Task Manager or reboot the PC.
3. If still unstable, use the restore point you created before registry edits.

For this specific tweak, restoring `System.IsPinnedToNameSpaceTree` back to `1` usually returns the default behavior.

![Clean File Explorer view](/images/_posts/windows11/onedrive-duplicate/onedrive-4.webp)
*My final File Explorer view, much better without those phantom folders.*

If you ever want to bring it back, just head back to Part 2 and change the value back to `1`. Hopefully, this helps you reclaim some visual peace in Windows 11!
