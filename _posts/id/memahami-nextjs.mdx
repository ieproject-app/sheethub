## Apa itu Server Components?
Server Components adalah fitur baru di Next.js yang memungkinkan Anda menulis komponen UI yang berjalan secara eksklusif di server. Hal ini dapat menghasilkan peningkatan kinerja yang signifikan dengan mengurangi jumlah JavaScript yang dikirim ke klien.

> Catatan: Server Components dapat langsung mengakses sumber daya sisi server seperti basis data atau sistem file tanpa perlu membuat rute API terpisah. Ini sangat menyederhanakan logika pengambilan data.

Berikut adalah contoh panjang seperti apa Server Component itu, yang mengambil data secara langsung:
```tsx
// src/app/page.tsx
import { db } from '@/lib/db';
 
// Ini adalah Server Component
async function HomePage() {
  const posts = await db.post.findMany();
 
  return (
    <main>
      <h1>Blog Saya</h1>
      <ul>
        {posts.map((post) => (
          <li key={post.id}>
            <a href={`/posts/${post.slug}`}>{post.title}</a>
          </li>
        ))}
      </ul>
    </main>
  );
}

export default HomePage;
```

## Kapan menggunakan Client Components?
Client Components adalah komponen yang sudah Anda kenal dari aplikasi React tradisional. Mereka dirender di klien dan dapat menggunakan state, efek, dan API khusus browser. Anda memilih untuk menggunakannya dengan menggunakan direktif `'use client'` di bagian atas file.

- Mereka dapat menggunakan `useState` dan `useEffect`.
- Mereka memiliki akses ke API browser seperti `window` atau `localStorage`.
- Mereka ideal untuk elemen UI yang interaktif.

> Ingatlah bahwa Anda hanya boleh menggunakan Client Components saat Anda benar-benar membutuhkannya. Mendorong sebanyak mungkin logika ke server adalah cara Next.js.
