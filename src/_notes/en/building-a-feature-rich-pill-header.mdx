Building a simple header is straightforward, but creating one that is both aesthetically pleasing and highly functional can be a complex journey. In this note, we'll walk through the process of building the animated "pill header" for this site, covering its core features and the interesting bugs we squashed along the way.

The goal was to create a compact, floating header that felt modern and integrated, rather than a static bar at the top of the page.

![A screenshot of the final, polished pill header on the homepage.](https://placehold.co/800x450/374151/fafafa?text=Screenshot+of+Final+Header)

### Core Features

We packed several key features into this small component:

*   **Expanding Search:** Clicking the search icon smoothly transforms the pill into a full-width search bar, providing an intuitive and clean user experience.
*   **Integrated Custom Menu:** Instead of a standard, detached dropdown, the menu "grows" directly from the pill header. We achieved this by dynamically changing the header's border-radius and using CSS transitions to create a seamless effect.
*   **Dynamic Visibility:** The header cleverly hides on scroll-down and reappears on scroll-up, maximizing screen real estate without getting in the way.
*   **Language Switcher:** A compact toggle allows users to switch between English and Indonesian content seamlessly.

### The Bug Hunt: Lessons Learned

No development process is complete without a few bugs. Here are some of the most notable ones we fixed:

1.  **The Flawed Language Switcher:** Initially, the logic for switching languages was too simplistic, causing it to fail on article pages. It would incorrectly modify the URL slug (e.g., `panduan-validasi` became `panduan-valenasi`), leading to 404 errors. The fix involved completely rewriting the logic to properly parse URL segments and handle translations correctly.

2.  **Responsiveness Nightmares:** Early versions had significant layout issues on mobile devices. The header and search results would often overflow the screen. This was traced back to conflicting Tailwind CSS classes (`w-full` combined with `left` and `right` properties), which we resolved by simplifying the responsive styles.

3.  **Search Result UI Glitches:** The search result dropdown was a hotbed of small but annoying visual bugs.

    *   **The Duplicate 'X' Button:** For a short time, two close buttons would appear when the search was active on mobile. This was a conditional rendering glitch that required more explicit rules to ensure only one was visible at any time.
    *   **Right-Side Clipping:** On mobile, the scrollbar would appear on top of the search results, cutting off the badge on the right. We fixed this by modifying the `ScrollArea` component itself to add internal padding, making space for the overlay scrollbar.
    *   **Text Wrapping Issues:** Long article titles in the search results refused to wrap, instead pushing the content out of its container. The culprit was a `truncate` class. We replaced it with `line-clamp-2` to allow titles to span two lines before being cut off, which fixed the layout beautifully.
