Kami menghadapi masalah CSS yang menarik di mana blok kode kami, yang diberi gaya menggunakan pustaka Shiki, tidak mendapatkan `padding` atau sudut membulat yang benar. Teks di dalamnya jadi terlalu mepet ke tepi, dan sudut yang tajam tidak sesuai dengan estetika situs kami.

Ini adalah kasus klasik "perang" spesifisitas CSS, atau terkadang hanya kelalaian sederhana dalam cara komponen disusun dan diberi gaya dalam kerangka kerja modern seperti Next.js.

### Usaha Awal: CSS Global

Pikiran pertama kami adalah menerapkan gaya global di `src/app/globals.css`. Ini biasanya merupakan pendekatan utama untuk penataan gaya umum.

```css
pre.shiki {
  @apply rounded-lg p-6 my-6 overflow-x-auto;
}
```

Secara teori, ini seharusnya berhasil. Kami menargetkan semua elemen `<pre>` dengan kelas `shiki`. Namun, entah mengapa, gaya tersebut tidak diterapkan.

### Solusi Akhir: Merender Komponen Secara Langsung

Setelah pendekatan global gagal, kami memutuskan untuk mengambil kendali yang lebih langsung. Dalam kerangka kerja berbasis komponen, cara paling andal untuk menata sesuatu sering kali adalah dengan mengontrol proses renderingnya secara langsung.

Kami memodifikasi `src/components/mdx-components.tsx`. File ini memberi tahu aplikasi kami cara merender elemen Markdown. Dengan mendefinisikan komponen khusus untuk tag `<pre>`, kami dapat secara paksa menerapkan gaya yang kami inginkan menggunakan kelas Tailwind.

```tsx
// src/components/mdx-components.tsx

// ... komponen lain

const MdxPre = ({ className, ...props }: React.HTMLAttributes<HTMLPreElement>) => (
    <pre
        className={cn(
            "rounded-lg p-6 my-6 overflow-x-auto",
            className
        )}
        {...props}
    />
);

export const mdxComponents: MDXComponents = {
  // ... komponen lain
  pre: MdxPre,
}
```

Dengan memetakan elemen `<pre>` secara eksplisit ke komponen `MdxPre` kami, kami menjamin bahwa setiap blok kode yang dirender dari MDX akan memiliki `padding`, sudut membulat, dan perilaku `overflow` yang benar. Masalah terpecahkan!
