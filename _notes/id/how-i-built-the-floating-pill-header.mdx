Jika Anda telah menjelajahi situs ini, kemungkinan besar Anda telah berinteraksi dengan header "pil" yang melayang di bagian atas halaman. Komponen ini dirancang minimalis namun kuat, menampung navigasi, pencarian, dan fitur daftar bacaan dalam satu komponen yang ringkas dan responsif. Dalam catatan ini, saya akan membedah bagaimana header ini dibangun menggunakan Next.js, React Hooks, dan Tailwind CSS.

### Teknologi Utama

- **Next.js App Router:** Untuk struktur komponen dan rendering sisi server.
- **React Hooks:** Terutama `useState`, `useEffect`, dan `useRef` untuk mengelola state, efek samping, dan referensi DOM langsung.
- **Tailwind CSS:** Untuk semua penataan gaya, memungkinkan desain yang cepat dan mengutamakan utilitas. Kami juga menggunakan `clsx` untuk penerapan class secara kondisional.
- **Lucide React:** Untuk ikon-ikonnya.

### Langkah 1: Struktur Dasar & Posisi Melayang

Pondasinya adalah elemen `<header>` dengan `position: fixed`.

```jsx
<header className="fixed top-4 left-4 right-4 md:left-1/2 md:-translate-x-1/2 ... z-50">
  <nav className="relative ... rounded-full bg-primary/90 backdrop-blur-sm ...">
    {/* Konten di sini */}
  </nav>
</header>
```
- `fixed top-4`: Ini menempelkan header di dekat bagian atas viewport.
- `left-4 right-4`: Pada perangkat seluler, header mencakup sebagian besar lebar layar.
- `md:left-1/2 md:-translate-x-1/2`: Pada layar sedang ke atas, header memposisikan dirinya di tengah secara horizontal.
- `bg-primary/90 backdrop-blur-sm`: Ini menciptakan efek kaca buram yang semi-transparan.
- `rounded-full`: Ini memberikan bentuk "pil".

### Langkah 2: Otak - Manajemen State Tampilan

Bagian paling krusial dari header adalah mengelola apa yang sedang ditampilkan. Saya menggunakan satu hook `useState` untuk ini:

```tsx
type ActiveView = 'none' | 'search' | 'menu' | 'readingList';
const [activeView, setActiveView] = useState<ActiveView>('none');
```

Setiap elemen interaktif di header cukup memanggil `setActiveView` untuk mengubah apa yang ditampilkan. Misalnya, mengklik ikon pencarian akan menyetel tampilan ke `'search'`, yang pada gilirannya memicu rendering kondisional untuk input pencarian dan panel hasil.

Bentuk dan gaya komponen juga bereaksi terhadap state ini. Saat `isSearchOpen` bernilai true, lebar elemen `nav` akan melebar. Saat `isMenuOpen` bernilai true, `border-radius` berubah dari `rounded-full` menjadi `rounded-t-2xl` untuk menyambung dengan panel dropdown di bawahnya.

### Langkah 3: Pengalaman Pencarian Dinamis

Fungsi pencarian adalah fitur utama.
1.  **Transisi State:** Mengklik ikon pencarian menyetel `activeView = 'search'`.
2.  **Animasi Kondisional:** Class CSS dialihkan menggunakan `clsx`. Tautan navigasi utama memudar (`opacity-0`) sementara kontainer input pencarian muncul (`opacity-100`).
3.  **Auto-focus:** Hook `useEffect` dipicu saat tampilan pencarian menjadi aktif, secara otomatis memfokuskan elemen `<input>` untuk pengalaman pengguna yang mulus.
    ```tsx
    const searchInputRef = useRef<HTMLInputElement>(null);

    useEffect(() => {
      if (isSearchOpen) {
        // Timeout singkat memastikan elemen terlihat sebelum difokuskan
        setTimeout(() => searchInputRef.current?.focus(), 100);
      }
    }, [isSearchOpen]);
    ```
4.  **Penyaringan Real-time:** `useEffect` lainnya memantau perubahan pada state `query` pencarian. Ini menyaring array yang berisi semua postingan dan catatan untuk menunjukkan hasil yang relevan secara instan.

### Langkah 4: Visibilitas Pintar saat Scroll

Pola UX yang umum adalah menyembunyikan navigasi saat scroll ke bawah dan menampilkannya saat scroll ke atas. Ini dicapai dengan hook `useEffect` lainnya.

```tsx
const lastScrollY = useRef(0);
const [isVisible, setIsVisible] = useState(true);

useEffect(() => {
  const handleScroll = () => {
    const currentScrollY = window.scrollY;
    // Tampilkan jika scroll ke atas atau di dekat bagian atas
    if (currentScrollY < lastScrollY.current || currentScrollY < 10) {
      setIsVisible(true);
    } else if (currentScrollY > 100 && currentScrollY > lastScrollY.current) {
      // Sembunyikan jika scroll ke bawah dan melewati ambang batas tertentu
      setIsVisible(false);
    }
    lastScrollY.current = currentScrollY;
  };

  window.addEventListener('scroll', handleScroll, { passive: true });
  return () => window.removeEventListener('scroll', handleScroll);
}, []);
```
State `isVisible` kemudian menambah atau menghapus class yang mengontrol `opacity` dan `transform` header.

### Langkah 5: Detail Kecil

- **Tombol Escape:** Event listener global disiapkan untuk menutup tampilan aktif apa pun (pencarian, menu, dll.) saat tombol 'Escape' ditekan.
- **Klik di Luar:** Listener serupa menutup tampilan jika pengguna mengklik di mana pun di luar komponen header.
- **Responsivitas:** Komponen bertransisi dengan anggun dari header seluler lebar penuh ke header desktop yang ringkas dan terpusat menggunakan prefix responsif Tailwind (`md:`).
