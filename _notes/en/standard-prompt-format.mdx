To ensure every development request or bug fix proceeds smoothly and efficiently, let's use this standard format. This helps me to fully understand your needs and avoid mistakes or overlooking important details.

### 1. For New Feature Requests

Use this format when you want to add new functionality.

{/*
```
**Feature Goal:**
[Briefly explain the ultimate goal of this feature. Example: "I want users to be able to save articles to read later."]

**Analysis & Concept (User Flow):**
[Describe how you envision a user interacting with this feature, step by step. Example: "1. On an article page, there is a 'Save' button. 2. After clicking, the article is added to a list. 3. In the header, there's an icon to view the saved list."]

**Visuals & Placement:**
[Describe where the UI components for this feature should appear. If possible, mention components from ShadCN or other existing libraries. Example: "The 'Save' button is under the article title. For the list, use the 'Sheet' component from ShadCN that slides in from the right when the header icon is clicked."]

**Technical Considerations (If Any):**
[If you have any technical preferences, mention them here. Otherwise, just leave it blank. Example: "For data storage, let's just use localStorage in the browser, no database needed."]
```
*/}

### 2. For Bug Reports or Visual Fixes

Use this format when you find something that isn't working or looking as it should.

{/*
```
**Problem Description:**
[Describe the issue you're seeing as clearly as possible. Example: "The article title in the search results is too long and overflows its box."]

**Problem Location:**
[Specify the page or specific component where this issue occurs. Example: "This happens inside the search results pop-up from the header."]

**Expected Behavior:**
[Describe what the correct appearance or functionality should be. Example: "A long title should wrap to a second line, and only if it's still too long should it be truncated with '...'.]

**Attachments (Highly Recommended):**
[Include screenshots or a short video. This greatly speeds up the process of identifying and fixing the issue.]
```
*/}

By following this format, we can ensure all details are communicated well and the development process runs more smoothly. Thank you for your cooperation!
