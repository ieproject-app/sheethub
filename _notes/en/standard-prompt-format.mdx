To ensure I can process your requests accurately and quickly, let's use this standard format. This will help me understand every detail and avoid mistakes or oversights.

### 1. **Main Goal**
*Briefly describe the final outcome you want.*

**Example:** "I want to add a new user profile page."

---

### 2. **Type of Change**
*Choose one: New Feature / Bug Fix / Content Change / Style Change.*

**Example:** "New Feature"

---

### 3. **Detailed Description**

*This is the most important part. Provide as much detail as possible.*

**For a NEW FEATURE or STYLE CHANGE:**
- **Functional Description:** How should this feature work? What happens when a user clicks something?
- **Visual Description:** What should it look like? Where is it located? What colors, sizes, or icons should be used?
- **Relevant Files (if you know):** Which files do you think need to be changed? (e.g., `src/components/layout/header.tsx`)

**For a BUG FIX:**
- **Reproduction Steps:** Describe the steps I need to take to see the bug.
- **Expected Behavior:** What should happen if there were no bug?
- **Actual Behavior (Bug):** What is actually happening? (Include a screenshot if possible).

**For a CONTENT CHANGE (New Article/Note):**
- **Article Title:**
- **Publication Date:** (e.g., 2026-02-16)
- **Publication Status:** `published: true` (to make it appear immediately) or `false`.
- **Language:** Does it need to be multi-lingual? If so, provide the content for each language.
- **Content Body:** Write the entire article content here.
- **Special Markers:** Tell me where to place markers for images, like: `{/* Place header screenshot here */}`

---

### 4. **Complete Request Example (for our Header Bug)**

Here’s an example of how you could have submitted the `line-clamp` bug fix we just solved, using this format.

*   **Main Goal:** Fix the appearance of article titles in the search results so they don't get cut off awkwardly.
*   **Type of Change:** Bug Fix.
*   **Detailed Description:**
    *   **Reproduction Steps:**
        1. Open the site in mobile view.
        2. Click the search icon.
        3. Type "header".
        4. Look at the search results that appear.
    *   **Expected Behavior:** If an article title is too long, the text should wrap to a second line. If it's still too long, then it should be truncated with an ellipsis (...).
    *   **Actual Behavior (Bug):** Long titles are forced into a single line and then truncated, making them unreadable and look bad. (Accompanied by the image you sent).
    *   **Relevant Files:** `src/components/layout/header.tsx`
