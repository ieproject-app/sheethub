We encountered an interesting CSS issue where our code blocks, which are styled using the wonderful Shiki library, were not getting the correct padding or rounded corners. The text inside was uncomfortably close to the edges, and the sharp corners just didn't fit our sites aesthetic.

This is a classic case of CSS specificity wars, or sometimes just a simple oversight in how components are structured and styled in a modern framework like Next.js.

### The Initial Attempt: Global CSS

Our first thought was to apply a global style in `src/app/globals.css`. This is usually the go-to for general styling.

```css
pre.shiki {
  @apply rounded-lg p-6 my-6 overflow-x-auto;
}
```

Theoretically, this should have worked. We were targeting all `<pre>` elements with the class `shiki`. However, for some reason, the styles were not being applied.

### The Winning Solution: Direct Component Rendering

After the global approach failed, we decided to take more direct control. In a component-based framework, the most reliable way to style something is often to control its rendering directly.

We modified `src/components/mdx-components.tsx`. This file tells our application how to render Markdown elements. By defining a custom component for the `<pre>` tag, we could forcefully apply the styles we wanted using Tailwind classes.

```tsx
// src/components/mdx-components.tsx

// ... other components

const MdxPre = ({ className, ...props }: React.HTMLAttributes<HTMLPreElement>) => (
    <pre
        className={cn(
            "rounded-lg p-6 my-6 overflow-x-auto",
            className
        )}
        {...props}
    />
);

export const mdxComponents: MDXComponents = {
  // ... other components
  pre: MdxPre,
}
```

By explicitly mapping the `<pre>` element to our `MdxPre` component, we guarantee that every code block rendered from MDX will have the correct padding, rounded corners, and overflow behavior. Problem solved!
